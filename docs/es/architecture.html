Detalles de los Componentes

1. Content Script (src/content/index.ts)

Este script se inyecta en la página de Google Meet. Sus principales responsabilidades son:

1.1 Motor de Análisis de Audio

Utiliza el AnalyserNode de la Web Audio API para medir el volumen de entrada del micrófono en tiempo real. 

getUserMedia() → AudioContext → MediaStreamSource → AnalyserNode
                                                          │
                                                    getFloatTimeDomainData()
                                                          │
                                                    Cálculo RMS (Root Mean Square)

Parámetros de configuración:

Parámetro Valor Descripción
fftSize 512 Tamaño de la ventana FFT
smoothingTimeConstant 0.3 Coeficiente de suavizado temporal
echoCancellation true Activar cancelación de eco
noiseSuppression false Supresión de ruido desactivada (para precisión del VAD)

1.2 Máquina de Estados

Una máquina de estados finitos con 5 estados gestiona el estado del habla.

IDLE ──(modo distinto de Off seleccionado)──► MUTED ──(volumen > speechThreshold)──► UNMUTING ──► SPEAKING
                     ▲                                                │
                     │                              (volumen < silenceThreshold)
                     │                                                │
                     └──(graceTimer expirado)── GRACE ◄───────────────┘
                                                │
                                         (volumen > silenceThreshold)
                                                │
                                                ▼
                                            SPEAKING

* Si el usuario alterna manualmente el micrófono de Meet, el modo permanece igual y solo el estado se actualiza.
Estado Descripción Acción en Meet
IDLE Modo Off o no inicializado Ninguna
MUTED Silencio detectado → silenciado Silenciar
UNMUTING Habla detectada → activando micrófono Activar micrófono
SPEAKING Hablando actualmente Ninguna
GRACE Habla finalizada → período de gracia Ninguna

1.3 Modos de Operación

Cuatro modos controlan cómo se comportan las transiciones de estado. Off es el modo con control deshabilitado.

Modo Activar micrófono Silenciar Atajo
Off Ninguno Ninguno Deshabilitado
Auto Automático al detectar voz Automático en silencio Alternar
Auto-Off Manual / atajo Automático en silencio Alternar
Push-to-Talk Mientras se mantiene la tecla presionada Tecla liberada → Gracia → Silenciar Mantener

1.4 Tecla de Atajo

Proporciona un atajo de teclado personalizable (por defecto: Ctrl+Shift+M).

  • Off: Deshabilitado
  • Auto: Alternar silencio
  • Auto-Off: Alternar silencio
  • Push-to-Talk: Activar micrófono solo mientras se mantiene la tecla (operación de mantener)

La tecla de atajo se puede cambiar libremente desde los ajustes del popup.

1.5 Diseño de Umbrales Asimétricos

Diferentes umbrales para activar y silenciar el micrófono previenen el chattering (alternancia frecuente). 

speechThreshold (inicio del habla)    = 0.025  (por defecto)
silenceThreshold (detección silencio) = speechThreshold × 0.5 = 0.0125

  Volumen
   ▲
   │  ┄┄┄┄┄┄┄┄┄ speechThreshold (0.025) ┄┄┄┄  ← Activar micrófono al superar
   │
   │  ┄┄┄┄┄┄┄┄┄ silenceThreshold (0.0125) ┄┄  ← Período de gracia al bajar
   │
   └──────────────────────────────────────────► Tiempo

1.6 Detección del Botón de Silencio

Detecta el botón de silencio de Google Meet usando múltiples estrategias:

  1. Búsqueda por atributo aria-label (compatible con japonés/inglés)
  2. Búsqueda por atributo data-tooltip
  3. Alternativa: simular atajo de teclado Ctrl+D

El estado de silencio se determina a partir del atributo data-is-muted o del contenido de texto de aria-label.

2. Popup UI (src/popup/)

Una interfaz de configuración construida con React 19. Sincroniza el estado con el Content Script en tiempo real a través de la Chrome Storage API.

  • Selector de modo: Cambiar entre Off / Auto / Auto-Off / Push-to-Talk
  • Indicador de estado: Muestra el estado actual (silenciado/hablando/gracia) con un indicador de punto
  • Medidor de nivel: Muestra el nivel de entrada del micrófono en tiempo real (con marcadores de umbral)
  • Control de sensibilidad: Ajustar speechThreshold (0.005–0.25) — guardado por modo
  • Control de período de gracia: Ajustar gracePeriod (500ms–4000ms) — guardado por modo
  • Configuración de tecla de atajo: Personalización mediante interfaz de grabación de teclas
  • Selección de dispositivo de micrófono: Selección manual del micrófono a utilizar

Flujo de Datos

Popup → Chrome Storage → Content Script
  │                            │
  │  hushMeetConfig: object    │  (ajustes de sensibilidad/período de gracia)
  │  hushMeetMode: string      │  (modo de operación)
  │  hushMeetShortcutKey: str  │  (tecla de atajo)
  │                            │
  │                            ▼
  │                      Content Script → Chrome Storage → Popup
  │                            │
  │                            │  hushMeetState: string  (estado actual)
  │                            │  hushMeetLevel: number  (nivel de volumen)
  │                            │  hushMeetSpectrum: array (espectro)
  └────────────────────────────┘

El Popup y el Content Script no se comunican directamente; se comunican bidireccionalmente a través de eventos de cambio de la Chrome Storage API.

Limitaciones Conocidas

  • La detección de voz se basa en RMS (volumen), por lo que puede activarse erróneamente con ruido ambiental fuerte (mitigado usando el modo Auto-Off)
  • Los cambios en la interfaz de Google Meet pueden romper la detección del botón de silencio
  • El inicio del habla puede cortarse (ajustable mediante la configuración de umbral)
  • La tecla de atajo solo funciona cuando la pestaña de Google Meet está enfocada

Mejoras Futuras

  • Puerta de ruido ponderada por voz para mejorar la precisión de detección
  • Almacenamiento previo al habla para reducir el recorte del inicio de la locución
  • Lógica de autodetección para seguir los cambios de la interfaz de Google Meet
  • Reemplazo con Silero VAD (basado en ML) para mayor precisión de detección
