Détails des composants

1. Content Script (src/content/index.ts)

Ce script est injecté dans la page Google Meet. Ses principales responsabilités sont :

1.1 Moteur d'analyse audio

Utilise l'API Web Audio AnalyserNode pour mesurer le volume d'entrée du microphone en temps réel. 

getUserMedia() → AudioContext → MediaStreamSource → AnalyserNode
                                                          │
                                                    getFloatTimeDomainData()
                                                          │
                                                    Calcul RMS (Root Mean Square)

Paramètres de configuration :

Paramètre Valeur Description
fftSize 512 Taille de la fenêtre FFT
smoothingTimeConstant 0.3 Coefficient de lissage temporel
echoCancellation true Activer l'annulation d'écho
noiseSuppression false Suppression du bruit désactivée (pour la précision du VAD)

1.2 Machine à états

Une machine à états finis à 5 états gère l'état de la parole.

IDLE ──(mode autre que Off sélectionné)──► MUTED ──(volume > speechThreshold)──► UNMUTING ──► SPEAKING
                     ▲                                                │
                     │                              (volume < silenceThreshold)
                     │                                                │
                     └──(graceTimer expiré)── GRACE ◄─────────────────┘
                                                │
                                         (volume > silenceThreshold)
                                                │
                                                ▼
                                            SPEAKING

* Si l'utilisateur bascule manuellement le microphone Meet, le mode reste le même et seul l'état suit.
État Description Action Meet
IDLE Mode Off ou non initialisé Aucune
MUTED Silence détecté → micro coupé Couper le micro
UNMUTING Parole détectée → réactivation Réactiver le micro
SPEAKING En train de parler Aucune
GRACE Fin de parole → période de grâce Aucune

1.3 Modes de fonctionnement

Quatre modes contrôlent le comportement des transitions d'état. Off est le mode de contrôle désactivé.

Mode Réactivation Coupure Raccourci
Off Aucune Aucune Désactivé
Auto Automatique à la détection vocale Automatique au silence Bascule
Auto-Off Manuel / raccourci Automatique au silence Bascule
Push-to-Talk Tant que la touche est enfoncée Touche relâchée → Grâce → Coupure Maintenir

1.4 Touche de raccourci

Fournit un raccourci clavier personnalisable (par défaut : Ctrl+Shift+M).

  • Off : Désactivé
  • Auto : Basculer le micro
  • Auto-Off : Basculer le micro
  • Push-to-Talk : Réactiver uniquement tant que la touche est enfoncée (maintenir)

La touche de raccourci peut être librement modifiée depuis les réglages du popup.

1.5 Conception à seuils asymétriques

Des seuils différents pour la réactivation et la coupure empêchent le chattering (basculement fréquent). 

speechThreshold (début de parole)    = 0.025  (par défaut)
silenceThreshold (détection silence) = speechThreshold × 0.5 = 0.0125

  Volume
   ▲
   │  ┄┄┄┄┄┄┄┄┄ speechThreshold (0.025) ┄┄┄┄  ← Réactivation si dépassé
   │
   │  ┄┄┄┄┄┄┄┄┄ silenceThreshold (0.0125) ┄┄  ← Période de grâce si en dessous
   │
   └──────────────────────────────────────────► Temps

1.6 Détection du bouton Mute

Détecte le bouton mute de Google Meet à l'aide de plusieurs stratégies :

  1. Recherche par attribut aria-label (prend en charge le japonais et l'anglais)
  2. Recherche par attribut data-tooltip
  3. Solution de repli : simulation du raccourci clavier Ctrl+D

L'état mute est déterminé à partir de l'attribut data-is-muted ou du contenu textuel de l'aria-label.

2. Popup UI (src/popup/)

Une interface de réglages construite avec React 19. Synchronise l'état avec le Content Script en temps réel via l'API Chrome Storage.

  • Sélecteur de mode: Basculer entre Off / Auto / Auto-Off / Push-to-Talk
  • Affichage de l'état: Affiche l'état actuel (coupé/en train de parler/grâce) avec un indicateur à pastille
  • Indicateur de niveau: Affiche le niveau d'entrée du microphone en temps réel (avec marqueurs de seuil)
  • Curseur de sensibilité: Régler le speechThreshold (0.005–0.25) — sauvegardé par mode
  • Curseur de période de grâce: Régler le gracePeriod (500ms–4000ms) — sauvegardé par mode
  • Réglages du raccourci clavier: Personnalisation via l'interface d'enregistrement de touches
  • Sélection du microphone: Sélection manuelle du microphone à utiliser

Flux de données

Popup → Chrome Storage → Content Script
  │                            │
  │  hushMeetConfig: object    │  (réglages sensibilité/période de grâce)
  │  hushMeetMode: string      │  (mode de fonctionnement)
  │  hushMeetShortcutKey: str  │  (touche de raccourci)
  │                            │
  │                            ▼
  │                      Content Script → Chrome Storage → Popup
  │                            │
  │                            │  hushMeetState: string  (état actuel)

Le Popup et le Content Script ne communiquent pas directement ; ils communiquent de manière bidirectionnelle via les événements de changement de l'API Chrome Storage.

Limitations connues

  • La détection vocale est basée sur le RMS (volume), elle peut donc se déclencher par erreur en cas de bruit ambiant fort (atténué par l'utilisation du mode Auto-Off)
  • Les modifications de l'interface Google Meet peuvent casser la détection du bouton mute
  • Le tout début de la parole peut être coupé (ajustable via les réglages de seuil)
  • La touche de raccourci ne fonctionne que lorsque l'onglet Google Meet est au premier plan

Améliorations futures

  • Porte de bruit pondérée par la voix pour une meilleure précision de détection
  • Mise en mémoire tampon pré-parole pour réduire la coupure en début d'énoncé
  • Logique de détection automatique pour suivre les changements d'interface de Google Meet
  • Remplacement par Silero VAD (basé sur le ML) pour une précision de détection accrue
